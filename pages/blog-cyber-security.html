Cyber Security Basics Every Developer Must Know

6 min read Diah M
Back to Blog

Security is not a feature you bolt on at the end of a project — it's a mindset you adopt from day one. As both a developer and a cyber security analyst, I've seen firsthand how small oversights can open massive vulnerabilities. Here are the fundamentals every developer needs to understand to build safer software.

Never trust user input

This is rule number one. Every piece of data coming from a user — form fields, URL parameters, headers, cookies — should be treated as potentially malicious until proven otherwise. Validate and sanitise all inputs on the server side, always. SQL injection and cross-site scripting (XSS) attacks are still among the most common vulnerabilities, and both are entirely preventable with proper input handling.

Use HTTPS everywhere

Transmitting data over plain HTTP is like sending a postcard — anyone between you and the recipient can read it. HTTPS encrypts the connection using TLS, protecting your users' data in transit. With free certificates available via Let's Encrypt, there's no excuse not to use HTTPS on every project, even in development.

Authentication vs. authorisation

These two concepts are often confused but critically different. Authentication is verifying who someone is — checking their password or token. Authorisation is verifying what they're allowed to do — checking their permissions. Mixing these up leads to serious access control bugs. Always check both: confirm the user is logged in AND confirm they have permission to access the requested resource.

Keep dependencies updated

Outdated packages are one of the most overlooked attack vectors. Attackers actively scan for applications running known vulnerable library versions. Run dependency audits regularly and update promptly when security patches are released. Tools like npm audit make this easy — there's no reason to skip it.

Threat modelling

Before you write a single line of code for a new feature, ask: what could go wrong? Who might try to misuse this? What data is at risk? This process — threat modelling — helps you identify security issues during design rather than after a breach. It doesn't have to be complex; even 15 minutes of structured thinking can prevent serious vulnerabilities.

Security is a shared responsibility. Developers who understand the basics of cyber security write better, safer code — and give their users the protection they deserve. Start with these fundamentals and build from there.
